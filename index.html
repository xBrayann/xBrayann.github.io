Framework

OpenSpec

Framework open source para desarrollo guiado por especificaciones, diseñado para trabajar con asistentes de código con IA como Claude, Copilot o Codex.

📦

Qué es

Un sistema para guardar especificaciones dentro del repositorio del proyecto para que la IA tenga siempre contexto claro.

Antes de escribir código, se define exactamente qué se quiere construir.

Sitio oficial →
🎯

Problema que resuelve

Cuando programas con IA sin estructura:

  • Las instrucciones se pierden en el chat
  • El contexto se degrada con el tiempo
  • El modelo comete errores acumulativos

OpenSpec mantiene las especificaciones como archivos versionados dentro del repositorio.

🔄

Flujo típico

  • Crear spec
  • Crear propuesta de cambio
  • Generar tareas
  • La IA implementa el código
  • Archivar cambios

Estructura de ejemplo

openspec/
  specs/
  changes/
      add-feature/
          proposal.md
          tasks.md
          design.md

En resumen: OpenSpec es un sistema para planear features antes de que la IA escriba código.

Análisis

Prompts vs Specs

La diferencia entre un proyecto caótico y uno bien documentado empieza antes de escribir la primera línea de código.

Sin estructura

Solo Prompts

Ejemplo de prompt típico:

Create a website with Angular,
.NET and SQLite
that shows countries with charts

Problemas resultantes:

  • Cada prompt es diferente e inconsistente
  • La IA pierde contexto con el tiempo
  • El proyecto se vuelve difícil de mantener
  • Requisitos poco claros generan bugs
Con estructura

Usando Specs

Flujo estructurado:

Idea  →  Spec  →  Tasks  →  AI implementa

Beneficios obtenidos:

  • Sistema documentado y versionado
  • La IA entiende el contexto completo
  • Decisiones trazables y reproducibles
  • Onboarding fácil para nuevos devs
Caso práctico

Ejemplo práctico

Imagina que tu web crece y quieres agregar múltiples features nuevas al mismo tiempo.

🗺️

Sin specs

Vas pidiendo features una por una a la IA. El contexto se fragmenta, las decisiones se contradicen y el código se vuelve inconsistente.

📋

Con specs

Defines primero: mapa del mundo, buscador, filtros, estadísticas y comparador de países. La IA entiende el sistema completo desde el principio.

Guía de uso

¿Cuándo usar Specs?

No siempre es necesario aplicar Spec-Driven Development. Aquí tienes una guía rápida.

✅ Úsalo cuando...
  • El proyecto es grande o complejo
  • Trabajan varias personas en el equipo
  • Habrá muchos cambios e iteraciones
  • Hay agentes de IA programando código
  • Necesitas trazabilidad de decisiones
❌ No hace falta cuando...
  • Es una web simple o landing page
  • Script pequeño o de un solo uso
  • Prototipo rápido o exploración
  • Proyecto personal sin colaboradores
